
# DeskClock 0.1.02 功能介绍与使用说明

## 1. 程序简介

`DeskClock 0.1.02` 是一个运行在 `Cardputer ADV` 上的桌面信息面板程序，适用于日常值守、无线电通联辅助和时间天气信息查看。

程序基于 `UIFlow2` / MicroPython 运行，主要提供以下能力：

- 本地时间与 UTC 时间显示
- 日期与星期显示
- 实时天气温度显示
- 日出、日落时间显示
- 经纬度显示
- 6 位梅登黑德网格（Maidenhead Grid）显示
- 当前正在通联的呼号显示
- WiFi 自动连接 / 扫描 / 保存
- WS 呼号服务连接状态显示
- 设置页面
- 亮度切换、熄屏、查看电量、手动刷新


## 2. 主界面显示说明

主界面主要包含以下信息：

- 大号时间：本地时间（东八区）
- `UTC xx:xx`：UTC 时间
- 日期：如 `2026-07-09`
- 星期：如 `Wed`
- 状态栏：如 `WS OK`、`WS Off`、`Wx OK`、`Offline`
- 温度：天气接口返回的实时温度
- `Call:XXXX`：当前正在通联的呼号
- 6 位网格：根据当前使用的经纬度自动计算
- 纬度 / 经度：当前天气定位使用的坐标
- `Sunrise / Sunset`：当天日出、日落时间


## 3. 当前版本主要功能

### 3.1 时间功能

- 开机联网成功后自动执行 NTP 校时
- 本地时间固定按东八区显示
- 同时显示 UTC 时间

### 3.2 天气与天文信息

- 根据当前使用的经纬度拉取真实天气数据
- 显示实时温度
- 显示当天日出、日落时间
- 天气信息支持手动刷新和定时刷新

### 3.3 位置与网格

- 显示当前使用的纬度和经度
- 自动计算并显示 6 位梅登黑德网格
- 可使用固定坐标
- 可启用 GPS，以 GPS 坐标替代固定坐标

### 3.4 呼号显示与 WS 状态

- 通过 WebSocket 连接呼号事件源
- 主界面显示当前正在通联的呼号
- 状态栏显示连接状态：
  - `WS OK`：WebSocket 已连接
  - `WS Off`：WebSocket 未连接
  - `WS Try`：WebSocket 正在重连
  - `Offline`：设备当前未联网

### 3.5 WiFi 功能

- 开机优先尝试已保存的 WiFi
- 若保存的 WiFi 不可用，可进入扫描列表重新选择
- 支持输入密码并保存

### 3.6 屏幕与设备控制

- 支持 4 档亮度循环切换
- 支持熄屏 / 亮屏
- 支持查看当前电池电量


## 4. 按键功能说明

### 主界面按键

- `L`：切换亮度
  - 共 4 档：`100% / 75% / 50% / 25%`
- `ESC`：熄屏 / 再次按下亮屏
- `S`：进入设置页面
- `B`：显示当前电量
- `R`：手动刷新天气、温度、日出、日落
- `W`：手动重连 WS 呼号服务

### 设置页面按键

- `;`：上移选择
- `.`：下移选择
- `Enter`：编辑或进入当前项
- `ESC`：退出设置页面并返回主界面


## 5. 设置页面说明

设置页面当前包含以下项目：

- `City`
  - 填写城市名称
  - 目前主要作为配置项保存，天气实际按经纬度坐标拉取

- `Lat`
  - 固定纬度

- `Lon`
  - 固定经度

- `GPS`
  - `ON`：优先使用 GPS 获取坐标
  - `OFF`：使用手动填写的固定坐标

- `IP`
  - 呼号服务主机地址或 IP
  - 用于 WS 呼号显示连接

- `WiFi Setup`
  - 重新扫描并配置 WiFi

- `Back`
  - 返回主界面


## 6. WiFi 使用说明

### 开机流程

1. 开机后先尝试连接已保存的 WiFi
2. 若连接成功，自动继续：
   - NTP 校时
   - 天气拉取
   - WS 连接
3. 若连接失败，则提示进入 WiFi 配置

### 重新配置 WiFi

1. 进入设置页面
2. 选择 `WiFi Setup`
3. 扫描周边 WiFi
4. 选择目标网络
5. 如需密码则输入密码
6. 连接成功后自动保存


## 7. 天气与位置逻辑说明

当前版本天气数据来源于网络天气接口，按“当前使用的经纬度”查询：

- `GPS = OFF`
  - 使用设置页面中的固定纬度、固定经度

- `GPS = ON`
  - 优先尝试读取 GPS 坐标
  - 若 GPS 获取失败，则回退到固定经纬度

日出、日落时间与温度均按该坐标查询。


## 8. 呼号显示逻辑说明

主界面的红色 `Call:` 区域用于显示当前正在通联的呼号。

程序会监听 WS 事件，并只在识别到“当前正在说话/通联中的呼号事件”时更新显示。未检测到有效通联时，显示为：

- `Call:---`


## 9. 状态栏说明

主界面右上区域会显示短状态文本，常见含义如下：

- `Boot`：程序启动中
- `Clock OK`：NTP 校时成功
- `Clock Loc`：未完成联网校时，使用本地时钟
- `Wx OK`：天气拉取成功
- `Wx Off`：天气拉取失败
- `WS OK`：WS 已连接
- `WS Off`：WS 未连接
- `WS Try`：WS 正在重连
- `Offline`：网络未连接
- `BATxx%`：当前电量
- `BL 100%` / `BL 75%` / `BL 50%` / `BL 25%`：亮度档位提示


## 10. 加载方式

本程序为 UIFlow2 / MicroPython 脚本版本。

使用时可将主程序文件加载到 Cardputer ADV 对应的 UIFlow2 运行环境中：

- 主程序文件：
  - `cardputer_adv_dashboard_uiflow2.py`

建议在继续修改前，先保存一份已验证可运行的备份版本。


## 11. 当前版本特点

相对于前面的实验版，当前版本特点如下：

- 主界面闪烁已经明显减轻
- 时间与 UTC 刷新更稳定
- `R` 与 `W` 功能已分离
  - `R` 仅刷新天气
  - `W` 仅重连 WS
- 日期与星期的重绘问题已修复


## 12. 已知说明

当前版本是“稳定优先”的版本，适合作为后续继续优化的基础版本。

后续如需继续改进，建议优先方向为：

- 进一步减小时间区域刷新时的视觉跳动
- 优化天气错误提示
- 增加更明确的网络/接口失败诊断信息
- 在稳定前提下，再考虑局部区域的离屏刷新优化


## 13. 建议使用流程

1. 先完成 WiFi 配置
2. 在设置页填写固定坐标或打开 GPS
3. 填写呼号服务的 IP 地址
4. 返回主界面等待时间、天气、WS 状态更新
5. 需要刷新天气时按 `R`
6. 需要重连呼号服务时按 `W`
